INTRODUCTION

Security is a state of well-being of information and infrastructures in which the possibility of successful yet undetected theft, tampering, and disruption of information and services are kept low or tolerable

 

Security - Basic Components – What should typically be there in a secured environment.

  1. Confidentiality : is the concealment of information or resources.
  2. Integrity : refers to the trustworthiness of data or resources in terms of preventing improper and unauthorized changes.
  3. Availability : refers to the ability to use the information or resource desired.
  4. Authenticity : is the identification and assurance of the origin of information.
  5. Accountability: Actions of an entity to be traced uniquely to that entity (we must be able to trace a security breach to responsible party). This supports nonrepudiation, deterrence, fault isolation, intrusion detection and prevention, and after-action recovery and legal action
  6. Non-repudiation : offer of evidence that a party indeed is the sender or a receiver of certain information.

 

Types of Security Attacks

  1. Eavesdropping - Message Interception (Attack on Confidentiality)

Unauthorized access to information, Packet sniffers and wire-tappers, Illicit copying of files and programs

 

  1. Tampering With Messages (Attack on Integrity)

Stop the flow of the message, Delay and optionally modify the message, Release the message again.

 

  1. Fabrication of Message (Attack on Authenticity)

Unauthorized assumption of other’s identity. Generate and distribute objects under this identity

 

  1. Preventing Messages (Attack on Availability)

Destroy hardware (cutting fiber) or software, modify software in a subtle way, Corrupt packets in transit, Denial of service (DoS): Crashing the server, Overwhelm the server (use up its resource)

 

  • DOS attack sub-categories

 

  1. Ping of Death Attack

The Ping of Death is a large ICMP packet sent by a workstation to a target. The target receives the ping in fragments and starts reassembling the packet. The size of the packet, once it is reassembled, is too big for the buffer and overflows it. This causes unpredictable results, such as reboots or system hangs. Early implementations of TCP/IP were (could be) affected. Now, firewalls etc. could filter out such packets.

 

  1. SMURF Attack

Sends many spoofed ICMP echo requests (ping) to a broadcast address on a victim’s network. The victim’s (spoofed) machine gets too many responses from every host on the network, consuming all available bandwidth. Attack based on spoofing source IP address.

CRYPTOGRAPHY

 

Cryptography, a word with Greek origins, means 'secret writing'.

  • Related terms: Cipher - A secret or disguised way/method of writing.

                                 Encipher- Convert a message/text into a coded form (encrypt).

 

 Although in the past cryptography referred only to 'encryption' and 'decryption' of messages with secret 'keys', today it is defined as involving mechanisms:

  • Symmetric-key encipherment.
  • Asymmetric-key encipherment.
  • Hashing.
  • Digital Signatures.
  • Digital Certificates.

 

 While the use of cryptography for providing 'confidentiality' is typically expected, cryptographic techniques are inextricably woven into 'authentication', 'message integrity', 'nonrepudiation', and more.

 

  1. Symmetric-key encipherment.

 

Symmetric-key algorithms are algorithms for cryptography that use the same cryptographic keys for both the encryption of plaintext and the decryption of ciphertext. The keys may be identical, or there may be a simple transformation to go between the two keys.

 

Figure 01 : Symmetric Encryption

 

There are two broad classes of symmetric encryption techniques:

  • Block ciphers - process messages in blocks, process the input one block of elements (e.g., a block = 64 bits) at a time, producing an output block for each input block.
  • Stream ciphers - process the input elements (e.g., element= a byte or a bit or units larger than a byte) continuously, producing output one element at a time, as it goes along. A typical stream cipher encrypts plaintext one byte at a time.

 

Block ciphers

The simplest approach to multiple-block encryption is known as electronic codebook (ECB) mode, in which plaintext is handled 'b' bits at a time. Each block is encrypted using the same algorithm and the same encryption key (K).

Figure 02 : ECB mode

 

Modes in Block Cipher

  • ECB mode

ECB mode stands for Electronic Code Block Mode. It is one of the simplest modes of operation. In this mode, the plain text is divided into a block where each block is 64 bits. Then each block is encrypted separately. The same key is used for the encryption of all blocks. Each block is encrypted using the key and makes the block of ciphertext.

  • CBC Mode

CBC Mode stands for Cipher block Mode at the sender side; the plain text is divided into blocks. In this mode, IV(Initialization Vector) is used, which can be a random block of text. IV is used to make the ciphertext of each block unique.

  • CFB Mode

CFB mode stands for Cipher Feedback Mode. In this mode, the data is encrypted in the form of units where each unit is of 8 bits.

  • OFB mode

OFB Mode stands for output feedback Mode. OFB mode is similar to CDB mode; the only difference is in CFB, the ciphertext is used for the next stage of the encryption process, whereas in OFB, the output of the IV encryption is used for the next stage of the encryption process.

  • CTR Mode

CTR Mode stands for counter mode. As the name is counter, it uses the sequence of numbers as an input for the algorithm. When the block is encrypted, to fill the next register next counter value is used.

 

Stream ciphers

It typically encrypts plaintext one byte at a time. A stream cipher eliminates the need to pad a message to be an integral number of blocks. A key (K) is input to a pseudorandom bit generator that produces a stream of 8-bit random numbers (unpredictable without knowledge of the input key). keystream (output) is combined one byte at a time with the plaintext stream using the bitwise exclusive-OR (XOR) operation.

Figure 03 : Stream cipher

 

  1. Public Key Cryptography

Public-key cryptography, or asymmetric cryptography, is a cryptographic system that uses pairs of keys: public keys (which may be known to others), and private keys (which may never be known by any except the owner). The generation of such key pairs depends on cryptographic algorithms which are based on mathematical problems termed one-way functions. Effective security requires keeping the private key private; the public key can be openly distributed without compromising security.

Algorithms used in public key encryption:

  1. RSA
  2. Diffie-Hellman algorithm
  3. Elliptic Curve Diffie–Hellman (ECDH)

RSA

The idea of RSA is based on the fact that it is difficult to factorize a large integer. The public key consists of two numbers where one number is multiplication of two large prime numbers. And private key is also derived from the same two prime numbers. So, if somebody can factorize the large number, the private key is compromised. Therefore, encryption strength totally lies on the key size and if we double or triple the key size, the strength of encryption increases exponentially. RSA keys can be typically 1024 or 2048 bits long, but experts believe that 1024 bit keys could be broken in the near future. But till now it seems to be an infeasible task.

 

 

Diffie-Hellman algorithm

The Diffie–Hellman key exchange method allows two parties that have no prior knowledge of each other to jointly establish a shared secret key over an insecure channel. This key can then be used to encrypt subsequent communications using a symmetric-key cipher.

 

Elliptic Curve Diffie–Hellman (ECDH)

The Elliptic Curve Diffie-Hellman (ECDH), a variant of the Diffie Hellman, allows two parties that have no prior knowledge of each other to establish a shared secret key over an insecure channel

 

  1. Hashing

Data integrity or integrity refers to the trustworthiness of data or resources in terms of preventing improper and unauthorized changes. End-point authentication or authentication is the identification and assurance of the origin of information. These types of security services could be implemented using Hash functions.

A cryptographic hash function is an algorithm that takes an arbitrary amount of data input—a credential—and produces a fixed-size output of enciphered text called a hash value, or just “hash.” That enciphered text can then be stored instead of the password itself, and later used to verify the user.

(Cryptographic) Hash function algorithms

  • MD5 (message-digest algorithm) hash function widely used (RFC 1321, 1992)
  • SHA-1 (Secure Hash Algorithm) (1993) is also used.
  • SHA-2 (2001) and SHA-3 (2015).

 

  1. Digital signatures

Handwritten signature attests to the fact that the person (who signed) acknowledged and/or agreed with the document’s contents. In a digital world, one often wants to indicate the owner or creator of a document, or to signify one’s agreement with a document’s content. A digital signature is a cryptographic technique for achieving these goals.

Figure 04 : Digital Signature

 

Algorithms used to implement Digital Signatures.

  • Digital Signature Standard (DSS), Digital Signature Algorithm (DSA
  • Elliptic-curve cryptography (ECC)

 

  1. Public-Key Certificates (Digital certificates)

A certificate consists of a public key plus a user/entity ID of the key owner with the whole information/block signed by a 'trusted' third party. Typically, the third party is a Certification Authority (CA) that is trusted by the user community, such as a government agency or a financial institution or telecommunications company, or other trusted peak organization.

Can Sri Lanka glean any insights to digital security based on the experience of Singapore?

When considering the situation happen in Singapore (DDoS attack),that lesson can be apply to Sri Lankan digital security industry. So, let’s look at how it can be apply to protect Sri Lankan digital security.

 

  1. Protect from Ping of Death (PoD) attacks

This type of DDoS attack is based on Application layer.

 

  • CAPTCHAs

”CAPTCHA” also known as    Completely Automated Public Turing test to tell Computers and Humans Apart is one of the proposed security technique to protect PoD attacks. Basically by this CAPTCHA puzzle, there is a challenge-response as attempt to ensure that the response given by a human or not (by machine).So the any user that use the server machine(related to DDoS attack)must successfully solve the CAPTCHA before that user establishing a TCP connection with that server machine.

 

Figure 06 : Example of CAPTCHA

 

But there are some problems with this CAPTCHA,

  • There must be a patience for users (CAPTCHA test are less user friendly)
  • Some spammers use real users to solving CAPTCHA giving small amount of money
  • There are some image recognition techniques used to bread CAPTCHAs by present

 

Because of those problems of CAPTCHAs, it was not using present

 

  • ConnectionScore scheme

In this method basically, the server can measure different, different statistical attributes for its users and also their traffic during the normal condition (which there is no attack against). So, by that, the server can use that information as a reference to when an attack happens. This ConnectionScore scheme predicts the legitimacy of connections with high amount of probability. Considering attributes can be given as below,

  • Request rate and download rate
  • Uptime and downtime
  • Browsing behavior
  • Source IP address distribution
  • Arrival distribution rate of users

 

  1. Protect from SMURF attacks

This type of DDoS attack is based on Network layer. Below techniques can be used to prevent from SMURF attacks,

  • Deploy scrubbing or inline DDoS mitigation technology
  • Blocking directed broadcast traffic that coming into the network
  • Disable IP broadcast addresses at each network router and firewall
  • Use Cloudflare’s DDoS protection

CONCLUSION

As Sri Lankans there some basic ways for handle with digital security (these are common ways, not only focus on Singapore attack). Those can be given as below,

  • Make sure to implement regular back-ups of important business data and information
  • Make sure to download and install software updates and patches for the Operating Systems and applications that use. Because those updates ensure the secure configuration of all of the users’ systems
  • Use a good antivirus software
  • Put good passwords to user accounts (also it’s good if change the password time to time)
  • Assist with establishing a robust incident response and disaster recovery plan should the worst should happen
  • Constant monitoring of user’s networks
  • Secure mobile working by develop a mobile device action plan and install encryption and mobile device management
  • Be cautious of suspicious links (from websites or coming to email)
  • Reduce to use debit cards online (also make sure to prevent from saving the payment information)
  • Avoid entering from unknown websites
  • Be aware of sharing things in social media
  • Avoid unnecessary downloads
